**Luna AGI** is an experimental symbolic and metacognitive architecture designed to explore how an artificial system can learn, validate, remember and reason through explicit structures instead of opaque black-box behavior.

It is built around the idea that intelligence should not only answer, but also understand how it knows what it knows.

**Core ideas:**

- Symbolic and relational memory
- Human-in-the-loop validation
- Speech act classification
- Concept maturity tracking
- Relation graph reasoning
- Cognitive homeostasis
- Memory hygiene and corruption quarantine
- Reflective and metacognitive modules
- External LLM bridge for structured consultation

**What this project demonstrates:**

- Python system design
- Modular cognitive architecture
- Knowledge validation workflows
- Symbolic reasoning
- Long-term memory modeling
- Auditability and traceability
- Experimental AI research mindset

> Luna is not presented as a finished AGI product.  
> It is a research-driven architecture for studying symbolic learning, memory, validation and reasoning.

### 📈 LN Trading Engine — Algorithmic Trading & Execution Governance

**LN Trading Engine** is an algorithmic trading infrastructure for MT5 designed as both an operational engine and a research/audit laboratory.

The system is built around one central principle:

> A serious trading engine must know when not to trade.

Instead of executing every signal, the engine passes market candidates through multiple layers of scanning, ranking, persistence, validation, risk control, execution governance and forensic logging.

**Core capabilities:**

- MT5 market data integration
- Multi-asset scanning
- Symbol resolution by broker suffix
- Warm-up phase before live decisions
- Top-1 candidate selection pipeline
- Deep validation layer
- Risk and execution management
- Break-even and living thesis engine
- Runtime safety modes
- Dashboard and operational health monitoring
- Shadow trade logging
- Backtest and replay audit scripts

**System philosophy:**

- Scan before selecting
- Persist before validating
- Validate before executing
- Govern before risking capital
- Log everything, including rejected trades

**What this project demonstrates:**

- Algorithmic system design
- Risk-aware automation
- Runtime governance
- Observability
- Decision pipelines
- Execution safety
- Research tooling
- Complex system decomposition

> This project is presented as software engineering and systems research, not as financial advice or a promise of trading performance.
